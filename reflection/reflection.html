Issue

One issue I encountered while making this website was with creating urls to navigate between the different website pages. If accessing one of the tabs from a page other than the home page, one would get a 404 error. Looking at the url in the web browser revealed that the paths were being stacked on each other, rather than being refreshed. Because I set up the directory to have a folder for the code associated with each page, the urls needed to move up into the parent directories instead of further down nonexistent subdirectories. I troubleshooted this by reading through a Developer Mozilla HTML guide to creating hyperlinks. I found that the urls I created in the html files needed to be proceeded by two periods (..) to move back to the parent directory before accessing the sub-directories each page's code was under.

CSS Transition

I applied a time-scaling CSS transition to the images associated with each project on the home page. When hovering over the image, the image's size increases by 10%.

I chose this transition because it would help to emphasize the visual content of each project to further grab users' attention.

To implement this CSS transition, I first researched the different types of CSS transitions and transition properties supported, such as through Mozilla Developer and Josh W Comeau's CSS Transition guide. I then attempted to create this transition. After finding that the image was not centering in the larger container of each project, I looked on Stackoverflow to see if anyone else had encountered this problem, including this thread I think looked on Stackflow to see if anyone else had encountered a similar problem. The suggestions made didn't apply to my scenario, and after exploring with the inspector developer tools, I found that it had to do with the display flex and align-items properties of the overarching divs applied to the larger group of content including the image.
