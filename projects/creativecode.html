Creative Coding

A series of sketches utilizing the P5.js and P5.sound libraries to create visual and audio artworks.

Overview and Usage

The core functionality of the sketches is that users can view six artworks in full-webpage mode. Some of them have built in interactivity where the visualization and/or audio changes based on user input, such as their mouse position, with others will generate random visualizations each time they are opened.

The specifics of each sketch functionality are:

  1. Sketch One: A static blue-gradient organic-arch shaped piece of artwork is generated
  2. Sketch Two: A piece of artwork with partial circles is generated. The circles have randomly generated positions, sizes, angles, and colors. The background is also randomly generated. When the web-page is refreshed, another piece of artwork is randomly generated.
  3. Sketch Three: A infinite-loop piece of artwork is generated; a fluid pink-purple-blue wave moves across the screen without stop.
  4. Sketch Four: A fluid gradient arch piece of artwork is modified by users' mouse position. The arch lines move based on the x and y position of a user's mouse, and the color changes based on the x-position of their mouse. When the x-position of the mouse is low (on the left side of the screen), the artwork turns to a pink-purple. When the mouse is at a high x-position (on the right side fo the screen), the color turns to a blue.
  5. Sketch Five: An audio-based artwork where when a user clicks on the play button, Clair De La Lune starts playing and the audio player button lights up to a yellow color and the background lightens.
  6. Sketch Six: A visual and audio-based artwork where users can click anywhere on the screen to play and pause Gymnopedie 1. Users can move their cursor around the screen to increase and decrease the size of the bright blue circle in the middle of the screen. This circle controls the volume of the music. As the user manipulates the circle to decrease in size, the music volume goes down, and when they increase the circle size, the volume gets louder. The center circle's hue reflects the volume level, where when the circle is smaller and the music is quieter, the circle turns to a darker blue. When the circle gets bigger and the volume gets louder, the circle turns to a brighter blue.

Development Process

The steps I took for this project included:

  1. Setting up a P5.js web editor account to run the code involving P5.js library without having to set up my coding environment to access the library and run the code locally.
  2. Researched the functionalities supported by the P5.js library through the Reference page.
  3. I first ideated what each sketch would entail and then I then picked out functions from the Reference page that could be leveraged to created those visualizations.
  4. I first wrote the code for setting up the canvas elements and then wrote the code for draw() and any other functions that needed to be defined and called within the setup() and draw() functions.
  5. I tested each sketch along the way in the preview view of the p5.js web editor.
  6. I repeated this process of each of the six sketches, going in order from the first sketch to the 6 sketch.
  7. Once I got to Sketch Five, I transfered the code I had created in the p5.js web editor into javascript and html files within my HCDE 438 portfolio to access and run local audio files. This involved defining the head and body scripts of each html page associated with each sketch.
  8. I then set up the gallery page and used flex box properties to make the page layout responsive to window size.

Ideas and Future Work

Some ideas I might consider implementing in the future include:

  • Modifing the artworks to allow for user customization. For example, users could specify which colors they want displayed in the artwork. Alternative, users could choose from a set list of color schemes.
  • Supporting the download of the artworks as a background screen for one's computer or code.
  • Allowing users to choose between multiple audio files to play for the audio-based sketches.
  • Allowing users to toggle the interactivity of the sketches on and off. This means supporting all of the sketches in both static and interactive modes.
  • Modifying the sketches to have more of an overarching theme. For example, connecting them all with an organic, arch-like gradient theme for cohesion and consistency.
