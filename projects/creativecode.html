Creative Coding

A series of sketches utilizing the P5.js and P5.sound libraries to create visual and auditory artworks.

Overview and Usage

The core functionality of the sketches is that users can view six artworks in full-webpage mode. Some of them have built-in interactivity where the visualization and/or audio changes based on user input, such as their mouse position, while others will generate random visualizations each time they are opened.

The specifics of each sketch functionality are:

  1. Sketch One: A static blue-gradient organic-arch-shaped artwork is generated.
  2. Sketch Two: A piece of artwork with partial circles is generated. The circles have randomly generated positions, sizes, angles, and colors. The background is also randomly generated. When the web page is refreshed, another piece of artwork is randomly generated.
  3. Sketch Three: A infinite-loop piece of artwork is generated; a fluid pink-purple-blue wave moves across the screen without stopping.
  4. Sketch Four: A fluid gradient arch piece of artwork is modified by a user's mouse position. The arch lines move based on the x and y position of a user's mouse, and the color changes based on the x-position of their mouse. When the x-position of the mouse is low (on the left side of the screen), the artwork turns to a pink-purple. When the mouse is at a high x-position (on the right side of the screen), the color turns blue.
  5. Sketch Five: An audio-based artwork where when a user clicks on the play button, Clair De La Lune starts playing and the audio player button lights up to a yellow color and the background lightens.
  6. Sketch Six: A visual and audio-based artwork where users can click anywhere on the screen to play and pause Gymnopedie 1. Users can move their cursor around the screen to increase and decrease the size of the bright blue circle in the middle of the screen. This circle controls the volume of the music. As the user manipulates the circle to decrease in size, the music volume goes down, and the volume gets louder when they increase the circle size. The center circle's hue reflects the volume level; when the circle is smaller and the music is quieter, the circle turns to a darker blue. When the circle gets bigger and the volume gets louder, the circle turns to a brighter blue.

Development Process

The steps I took for this project included:

  1. Setting up a P5.js web editor account to run the code using the P5.js library without having to set up my coding environment to access the library and run the code locally.
  2. Researching the functionalities supported by the P5.js library through the Reference page.
  3. Ideating what each sketch would entail and then picking out functions from the Reference page that could be leveraged to create those visualizations.
  4. Writing the code for setting up the canvas elements and then writing the code for draw() and any other functions that needed to be defined and called within the setup() and draw() functions.
  5. Testing each sketch along the way in the preview view of the p5.js web editor.
  6. Debugging any errors in the code by using the console and inspecting the page with the web browser tools.
  7. I repeated this process for each of the six sketches, going in order from the first sketch to the sixth sketch.
  8. Once I got to Sketch Five, I transferred the code I had created in the p5.js web editor into javascript and html files within my HCDE 438 portfolio to access and run local audio files. This involved defining the head and body scripts of each html page associated with each sketch.
  9. I then set up the gallery page and used flexbox properties to make the page layout responsive to window size.

Issue Deep-Dive

One logic error I ran into involved the play function of the p5.sound library. For the last sketch, Sketch 6 (Audio and Visuals). When I first added the play() function to this sketch, I added it within the draw() function instead of mousePressed() function. This caused the animations that were previously responsive to break and the whole page became frozen. After talking to the professor, I learned that calling the play function in draw() can cause the page to break because of how many times the play() function would be called there. I then transferred the if statements involving the play() function to be within the mousePressed() function, which resolved the issue.

Ideas and Future Work

Some ideas I might consider implementing in the future include:

  • Modifying the artwork to allow for user customization. For example, users could specify which colors they want displayed in the artwork. Alternatively, users could choose from a set list of color schemes.
  • Supporting the download of the artworks as a background screen for one's computer or code.
  • Allowing users to choose between multiple audio files to play for the audio-based sketches.
  • Allowing users to toggle the interactivity of the sketches on and off. This means supporting all of the sketches in both static and interactive modes.
  • Modifying the sketches to have more of an overarching theme. For example, the sketches could be connected through an organic, arch-like gradient theme for cohesion and consistency.
