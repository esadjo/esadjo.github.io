Final Project Proposal

Breaking down my plans for my HCDE 438 Final Project deliverable

Project Description

My proposed final project is a browser extension that allows users to add a persistent spherical plant world to their screen, which reflects the current weather of an inputted location.

Note: This project will extend the plant web browser extension I made for MP1.

Prototypes & Versions of Project

There will be two versions of my project.

Prototype Version

This prototype will showcase the core features of the web browser extension. This includes users being able to input a specific city/location into a text input box and the weather conditions will be retrieved from the Weather API. To simplify the prototype, the location inputted will not be stored and remembered after closing the extension popup in this version. Users will also be able to select a plant from 3 options. Users can then add the plant world to their page through a toggle. At this stage, the background and the plant displayed will be hardcoded. The background will only display the day condition (the night condition is excluded). The plant displayed will be one of the 3 plants (1 of 3 images will be used). This is compared to the intended 12 images for plants (4 variations of each plant based on the weather condition). See the image below.

A breakdown of the 12 plant images that will be displayed in the plant world based on the weather conditions (see middle column). These are then either overlayed on top of the day or night versions of the background.

Final Version

For the final version of the web browser extension, users should be able to enter a location of their choice and select a plant type. The location entered is updated inline on the popup screen. If the user turns on the toggle, the plant world appears, consisting of 3 elements reflecting the weather conditions of the specified location: the background (day or night), the plant (1 of 12 based on the plant type previously selected and the current weather condition), and the weather icon (retrieved from the Weather API). Users' inputted location, plant type, and current display toggle state are remembered when the popup is closed and/or accessed across tabs. The weather conditions are retrieved from the API every 15 minutes and the three image elements of the plant world are updated accordingly to reflect the current weather conditions and time. All of this will be achieved without any hardcoded elements.

Plan for Development Process

Below is a step-by-step plan for my development process:

  1. Create a new repository by cloning my MP1 web browser extension GitHub repository as a starting point.
  2. Integrate dev toolchain (local development server, build tool, and deployment tool) into the repository.
  3. First add all the elements to the web browser popup. This includes adding headers, text, text input fields, toggles, etc. At this point, they will not be responsive.
  4. Work on the popup files (mainly popup.js) to save the location users input. The location should be saved and displayed on the popup for user feedback.
  5. Work on calling the Weather API to retrieve the current weather conditions of the location inputted by the user. This will be achieved through the Realtime API method. I will want to access and store the results for condition:text (Weather condition text), condition:icon (Weather icon url), and is_day (to know if it's day or night).
  6. Storing the data from the Weather API.
  7. Add all the visual elements of the plant world to a block element overlayed on top of the web pages. At this point, the visual elements do not need to be accurately positioned. They just need to be included. The three elements are the background of the world (sky and dirt), the plant, and the weather icon (which is retrieved from the weather API). The background and the plant will be hard coded, but the weather icon should be retrieved from the weather API and should reflect the weather of the location specified by the user.
  8. Milestone 1: Prototype complete. At this stage, I will begin the final version.
  9. Duplicate the Prototype 1 repository for the final version.
  10. Modify the popup.js and content script files to display the correct background and plant based on users' inputted location and selected plant type.
  11. Modify the popup.js file to store and remember users' inputs for all fields when they close the browser extension popup and/or move to a different web browser tab.
  12. Modify the content script files and popup.js file to allow the plant world to reflect whether or not it is day. This is so that the plant world will display either the day or night background (sky and dirt background).
  13. Modify the content script files to visually organize the plant world overlayed on top of web pages. The three elements of the plant world (the background, the plant, and the weather icon) should all be properly positioned, as displayed in the previous image above.
  14. Clean up and finalize the visual design of the popup. This includes the hierarchy, visual layout, typography, etc.
  15. Modify the popup.js and contentscript.js to regularly update (e.g., every 15 minutes) to call the Weather API to update the weather icon displayed in the world. This is so that the plant world displays the current weather.
  16. Milestone 2: Final version complete.

Known Unknowns

To be able to complete this project, I will have to fill the following knowledge gaps:

  • How to ensure that the Weather API is called only after a set interval of time (every 15 minutes) once the user input of the location is accepted. I will need to learn how to reduce the number of calls made if the user refreshes their browser several times before 15 minutes have passed without the location inputted changing. This is to reduce the number of API calls made because there is a limit.
  • How to make sure that the extension is only calling the API when the plant world is turned. I will need to ensure that it isn't accidentally calling the API in the background when the plant world is toggled off to reserve energy and ensure the API limit for the number of calls made isn't reached.
  • How to efficiently store multiple variables to be remembered after the popup is closed and while navigating across tabs.
  • How to add and position multiple images within a block element in the content script files.

Stretch Goals

My stretch goals include:

  • Establishing a counting system and warning message for when users are close to reaching their max number of "refreshes" of the browser because there is a limited number of calls to the API that can be made each month (1,000,000 per month).
  • Allowing users to save a list of locations for easy access.
  • Allowing users to change the size and location of the plant world.
  • Allowing users to 'plant' multiple plants in the plant world instead of just one.
  • Animating the plants to grow over time.
  • Animating the weather conditions instead of having them be static.
