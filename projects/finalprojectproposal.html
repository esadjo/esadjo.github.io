Final Project Proposal

Breaking down my plans for my HCDE 438 Final Project deliverable

Project Description

My proposed final project is a browser extension which allows users to add a persistent plant ecosystem to their screen that reflects the current weather of a selected location.

Note: This project will extend the plant web browser extension I made for MP1.

Prototypes & Versions of Project

Descriptions of two versions of your project:

Prototype Version

This prototype will showcase the core features of the web browser extension. This includes users being able to input a specific city/location into a text input box and the weather conditions retrieved from the Weather API. To simplify the prototype, the location inputted will not be stored and remembered after closing the extension popup. Users will also be able to select a plant from 3 options. Users can then add the plant world to their page through a toggle. At this stage, the background and the plant displayed will be hardcoded. The background will only display the day condition (the night condition is excluded). The plant displayed will be one of the 3 plants (1 of 3 images will be used). This is compared to the intended 12 images for plants (4 variations of each plant based on the weather condition). See the image below.

Display of the 12 plant images that will be displayed in the plant world based on the weather conditions. These are then either overlayed on top of the day or night version of the background.

Final Version

For the final version, users should be able to enter a location of their choice and select a plant type. The location entered is updated inline on the popup screen. If users turn on the toggle, the plant world appears and consistent of 3 responsive elements: the background (day or night), the plant (1 of 12 based on the plant type previously selected and the current weather condition), and the weather icon (retrieved from the weather API). Users' inputed location, plant type, and current display toggle state are remembered when the popup is closed and across tabs. The weather conditions are retrieved from the API every 30 minutes to an hour and 3 image elements of the plant world are updated to reflect the current weather conditions and time. All of this will be achieve without any hardcoded elements.

Plan for Development Process

Below is a step-by-step plan for my development process:

  1. Create new repository by cloning MP1 web browser extension GitHub repository as a starting point.
  2. First add all the elements to the web browser popup. This includes adding headers, text, text input fields, toggles, etc. At this point, they will be responsive
  3. Work on the popup files (mainly popup.js) to save the location users input. The location should be saved and displayed on the popup for user feedback.
  4. Work on calling the Weather API to retrieve the current weather conditions of the location inputed by the user. This will be achieve through the Realtime API methodI will want to access and store the results for condition:text (Weather condition text), condition:icon (Weather icon url), and is_day (to know if it's day or night).
  5. Storing the data from the Weather API.
  6. Add all the visual elements of the plant world to a block element overlayed on top of the web pages. At this point, the visual elements do not need to be accurately positioned, they just need to be included. The three elements are the background of the world (sky and dirt), the plant, and the weather icon (which is retrieved from the weather API). The background and the plant will be hard coded, but the weather icon should be retrieved from the weather API and should reflect the weather of the location specified by the user.
  7. Milestone 1: Prototype complete. At this stage, I will begin the final version.
  8. Modify the popup.js and content script files to display the correct background and plant based on user's inputed location and selected plant type.
  9. Modify the popup.js file to store and remember users' inputs for all fields when they close the browser extension popup and/or move to a different web browser tab.
  10. Modify the content script files and popup.js file to allow the plant world to reflect whether or not it is day or not. This is so that the plant world will display either the day or night background (sky and dirt background).
  11. Modify the content script files to visually organize the plant world overlayed on top of web pages. The three elements of the plant world (the background, the plant, and the weather icon) should all be properly positions as displayed in the previous image above.
  12. Clean up and finalize the visual design of the popup. This includes the hierarchy, visual layout, typography, etc.
  13. Modify the popup.js and contentscript.js to regularly update (e.g., every 30 minutes to an hour) to call the weather API to update the weather icon displayed in the world. This is so that the plant world displays the current weather.
  14. Milestone 2: Final version complete.

Known Unknowns

To be able to complete this project, I will have to fill the following knowledge gaps:

  • How to automatically call an API only after a given interval of time
  • How to store multiple variables to be remembered after the popup is closed and while navigating across tabs.
  • How to add and posiiton multiple images to a block element contentscript files.

Stretch Goals

My stretch goals include:

  • Allow users to a list of locations for easy access.
  • Allow users to change the size and location of the plant world.
  • Allow users to 'plant' multiple plants in the plant world instead of just one.
  • Animate the plants to grow overtime.
  • Animate the weather conditions instead of having them be static.
