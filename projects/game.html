(Dev Toolchain) Game

An dodge-the-fire balloon game created using libraries dia web development toochain

Overview and Usage

The core functionality of this project is that in the game, users control a blue balloon. Using the arrow controller keys on their keyboard, users can move the ballon left, right, up, and down at a set speed. If the balloon moves off the page, it appears on the other side (left to right side of screen, top to bottom of screen, and vice versa) to keep the player in the game. Meanwhile, fireballs rotating at different speeds move across the screen, which the user has to avoid. If the user's balloon gets hit by a fireball, then the balloon expands. If the score reaches 35 hits, the game is over.

Smaller functionalities included in this game are that the background is responsive. As the balloon acquires more hits and expands, the background red arches further shift to the right side of the screen. The game includes instructions in the center bottom of the screen to guide users at the beginning of the game and disappears after the users begins the game and has a score of 2 hits. Additionally, another functionality is that users can click anywhere on the screen to play or pause the game background music. By default, the music is off. The sound icon in the top right corner updates to show which sound state is selected.

Development Process

The steps I took for this project included:

  1. Following in-class activity to install libraries [ https://github.com/branchwelder/web-technologies/blob/main/activities/06_toolchain.md ]
  2. Reading up on P5.play functions at [ADD LINK]
  3. Going through examples in [ADD Link] and observing which P5.play functions they leveraged to create their interactions and animations. In the Kudos section, I specify which examples I studied in particular.
  4. Building out the most basic functionality and then working ways up. First getting the creating sprite for the player and having it controlled by the arrow keys. Then adding the fireballs and defining their moving behavior. Then writing the code to make the balloon expand when it overlaps with the fireballs. Then adding background effects like the background arches moving right as the number of hits increase. Then adding visual elements such as the score number of hits at the top, and then the sound arrow. I focused on working on the elements involving p5 and p5.play first, and then worked on adding the sound elements with p5.sound.
  5. Following in-class activity again to figure out how to set up live server to serve my code and to bundle it with Rollup. This included getting help via discord to understand instructions.

Ideas and Future Work

Some ideas I might consider implementing in the future include:

  • Allowing users to choose game diffculty level. This would be controled by the speed of the fireballs, how many fireballs there are, the size of the fireballs the speed of the balloon, etc.
  • Providing player customization: users being able to choose other player avatars besides a balloon, or being able to specify the color of the balloon.
  • Allowing users to customize the game setting by being able to choose between a selection of game backgrounds.
  • Allowing a life system and incrementing the difficulty of the game in several stages for a larger game journey.
